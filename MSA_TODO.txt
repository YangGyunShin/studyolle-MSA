    [전체 호출 흐름 — 다섯 단계 네트워크 호출]
      브라우저 fetch
        → admin-frontend POST /members/{id}/role
          → admin-frontend AdminInternalClient.updateMemberRole()
            → admin-service POST /api/admin/members/{id}/role
              → admin-service AccountAdminClient.updateRole() (Feign)
                → account-service POST /internal/accounts/{id}/role
                  → AccountInternalService.updateRole() — 검증 3종 + DB UPDATE

    [HTTP 메서드 결정]
    초안에서는 PATCH 를 선택했으나, RestTemplate 기본 클라이언트가 PATCH 미지원이라
    "Invalid HTTP method: PATCH" 에러 발생. 의존성 추가 대신 전 계층 POST 로 통일.

[x] 스터디 강제 비공개 — 완료 ✅ (2026-04-16)
    소스 작성 + HTML 템플릿 + 네비게이션 연결 전체 완료.
    상세 문서: 스터디_강제_비공개.md

    [설계 결정 — Option A]
    "강제 비공개" 의 최종 상태를 일반 close() 와 동일하게 맞춘다
    (closed=true, published 는 변경하지 않음). State Machine 불변식 유지.

    [전체 호출 흐름 — 다섯 단계 네트워크 호출]
      브라우저 fetch
        → admin-frontend POST /studies/{path}/force-close
          → admin-frontend AdminStudyClient.forceCloseStudy()
            → admin-service POST /api/admin/studies/{path}/force-close
              → admin-service StudyAdminClient.forceCloseStudy() (Feign)
                → study-service POST /internal/studies/{path}/force-close
                  → StudyInternalService.forceClose() — 검증 + 엔티티 상태 전이

    통합 테스트는 후속 작업.

    [설계 결정 — 옵션 B 하이브리드 (B1: 인증 시 JWT 재발급)]
    "이메일 인증 안 한 사용자도 로그인은 가능하지만, 쓰기 기능은 막는다"
