[1] study-service InternalStudyController 엔드포인트 추가
    StudyPageController 가 StudyInternalClient 를 통해 호출하는 엔드포인트들이
    현재 study-service 에 존재하지 않는다.
    스터디 페이지와 대시보드를 실제로 띄우려면 아래가 먼저 구현되어야 한다.

    추가 필요한 엔드포인트:
      GET  /internal/studies/{path}/page-data?accountId={id}
           → StudyPageDataResponse 반환
             (id, path, title, description, image, published, closed, recruiting,
              joinType, removable, memberCount, managers, members,
              manager(bool), member(bool), hasPendingRequest(bool))

      GET  /internal/studies/dashboard?accountId={id}
           → DashboardResponse 반환
             (studyManagerOf, studyMemberOf, studyEventsMap, enrolledEventIds, studyList)

      GET  /internal/studies/{path}/join-requests
           → List<JoinRequestResponse> 반환 (PENDING 상태만)

      GET  /internal/studies/{path}/tags-str
           → List<String> 반환 (태그 이름 목록)

      GET  /internal/studies/{path}/zones-str
           → List<String> 반환 (지역 이름 목록)

      GET  /internal/studies/tag-whitelist
           → List<String> 반환 (전체 태그 이름 목록, Tagify 자동완성용)

      GET  /internal/studies/zone-whitelist
           → List<String> 반환 (전체 지역 이름 목록)

      GET  /internal/studies/recent
           → List<StudySummaryResponse> 반환 (최근 공개된 스터디, 비로그인 랜딩용)

    모든 엔드포인트는 기존 InternalRequestFilter 적용 대상이므로
    X-Internal-Service 헤더 없으면 403 반환됨 (추가 작업 불필요).

    ※ DTO 클래스 4개 이전 세션에서 신규 생성 완료:
       StudyPageDataResponse.java, StudySummaryResponse.java,
       DashboardResponse.java, JoinRequestResponse.java
