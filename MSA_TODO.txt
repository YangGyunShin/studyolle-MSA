Phase 5 — notification-service + Redis
================================================

[공부 목적] Kafka + RabbitMQ + Redis 혼용 설계
  - study-service  → Kafka    → notification-service  (스터디 이벤트)
  - event-service  → RabbitMQ → notification-service  (모임/enrollment 이벤트)
  - notification-service → Redis (알림 카운터, 중복 처리 방지)
  - 배우는 것:
      Kafka:    Topic, Partition, Consumer Group, Offset, KafkaTemplate
      RabbitMQ: Exchange, Queue, Binding, Routing Key, RabbitTemplate
      Redis:    INCR/DECR 카운터, SETNX 중복방지, TTL
