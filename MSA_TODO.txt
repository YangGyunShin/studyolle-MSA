Phase 5 — notification-service
================================================

[공부 목적] Kafka + RabbitMQ 혼용 설계
  - study-service  → Kafka    → notification-service  (스터디 이벤트)
  - event-service  → RabbitMQ → notification-service  (모임/enrollment 이벤트)
  - 배우는 것:
      Kafka:    Topic, Partition, Consumer Group, Offset, KafkaTemplate
      RabbitMQ: Exchange, Queue, Binding, Routing Key, RabbitTemplate

[ ] Docker 기동
    docker run -d --name zookeeper -e ZOOKEEPER_CLIENT_PORT=2181 -p 2181:2181 confluentinc/cp-zookeeper:7.5.0
    docker run -d --name kafka \
      -e KAFKA_BROKER_ID=1 \
      -e KAFKA_ZOOKEEPER_CONNECT=host.docker.internal:2181 \
      -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \
      -e KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1 \
      -p 9092:9092 confluentinc/cp-kafka:7.5.0
    docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:3-management
    docker run -d --name notification-db -p 27017:27017 mongo:7

[ ] study-service — Kafka Producer 추가
    - 이벤트: 스터디 생성, 공개, 모집 시작/종료
    - Topic: study-events
    - 의존성: spring-kafka
    - KafkaTemplate<String, StudyEvent> 으로 발행

[ ] event-service — RabbitMQ Producer 추가
    - 이벤트: 참가 신청, 수락, 거절, 출석
    - Exchange: event.notification (Topic Exchange)
    - Routing Key: enrollment.accepted / enrollment.rejected 등
    - 의존성: spring-boot-starter-amqp
    - RabbitTemplate 으로 발행

[ ] notification-service 신규 생성 (:8085)
    - 기술 스택: MongoDB + Kafka Consumer + RabbitMQ Consumer
    - Kafka Consumer:     @KafkaListener(topics = "study-events")
    - RabbitMQ Consumer: @RabbitListener(queues = "enrollment.queue")
    - 알림 저장: MongoDB (Notification 도큐먼트)
    - API:
        GET   /api/notifications        읽지 않은 알림 목록
        PATCH /api/notifications         전체 읽음 처리
        PATCH /api/notifications/{id}    단건 읽음 처리

[ ] api-gateway 라우팅: /api/notifications/** → lb://NOTIFICATION-SERVICE
